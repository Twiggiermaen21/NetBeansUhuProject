Project Documentation: Messi GOAT GYM

Detailed technical reference for the gym management system, covering Controllers, DAOs, Views, and Entity Models.

Configuration & Utilities CONFIG

Infrastructure classes for database connectivity.

HibernateUtil.java

Method Signature Precise Functionality
buildSessionFactory(String user, String pass) Dynamically configures Hibernate. Overrides hibernate.connection.username, password, and url properties at runtime based on user input. Builds the ServiceRegistry and returns a SessionFactory.
getSessionFactory() Returns the static singleton instance of the session factory.
close() Safely shuts down the `SessionFactory` and destroys the `StandardServiceRegistry` to release JDBC resources.

Controller Layer (Logic) CONTROLLER

Orchestrates data flow between the GUI (View) and the Database (DAO).

MainController.java

The central nervous system of the application.

Method Signature Precise Functionality
MainController(SessionFactory sessionFactory) Constructor. Initializes the main view, instantiates sub-controllers (Tables), registers listeners, and sets the initial "Init" screen.
refreshActivityCombo() Fetches activities via DAO and populates the enrollment ComboBox. Uses a custom cell renderer to display getAName() instead of object hash codes.
initTableSelectionLogic() Adds a listener to the main table. When a Client row is selected, it auto-fills the "Client Name" field in the enrollment panel.
handleSaveEnrollment(String clientCode, Activity activity) Many-to-Many Logic: Fetches the Client and Activity entities, adds the Client to the Activity's clientSet, and commits the transaction.
actionPerformed(ActionEvent e) Handles navigation (Menu clicks). Switches views (Client/Trainer/Activity), updates button labels, and toggles panel visibility.
handleFormAction(Object entity, String type) Factory method for forms. Opens DataUpdateWindow and assigns the correct specific controller (Client/Trainer/Activity DataController) based on context.
deleteEntity(String type, String code) Executes deletion via the appropriate DAO based on the current view context and refreshes the table upon success.

ConnectionController.java

Method Signature Precise Functionality
appEntry() Retrieves credentials from ConnectionView. Calls HibernateUtil.buildSessionFactory. If successful, launches MainController; otherwise, shows error.
exitApplication() Calls HibernateUtil.close() and terminates the JVM (`System.exit(0)`).

ClientDataController.java

Handles the "Add/Edit Client" form logic.

Method Signature Precise Functionality
initializeForm() Configures form labels for Clients. If adding new: calls initializeFormWithAutoData. If editing: calls populateForm.
generateNextMemberNumber(String maxNum) Parses the highest current ID (e.g., "S005"), increments the integer part, and formats the new ID (e.g., "S006").
FormSubmitListener.actionPerformed Validation Logic: Checks required fields, DNI format (8 digits+Char), Email regex, and Age (>18). Checks ID uniqueness via DAO before saving.

TrainerDataController.java

Handles the "Add/Edit Trainer" form logic.

Method Signature Precise Functionality
generateNextTrainerCode(String maxNum) Generates sequential Trainer IDs (e.g., T001 -> T002).
FormSubmitListener.actionPerformed Validates inputs (ID format, non-future dates). Ensures the Trainer ID is unique in the DB before calling insertTrainer or updateTrainer.

ActivityDataController.java

Handles the "Add/Edit Activity" form logic.

Method Signature Precise Functionality
fillTrainerComboBox() Fetches all trainers and populates the dropdown, allowing the user to assign a trainer to the activity.
FormSubmitListener.actionPerformed Conflict Check: Calls activityDAO.isTrainerOccupied to ensure the selected trainer isn't already teaching at that Day/Time. Prevents double-booking.

Table Helpers (Client/Trainer/ActivityControllerTable)

Utils responsible for fetching lists and refreshing the main JTable.

Method Signature Precise Functionality
showClients() / showTrainers() / showActivities() Executes HQL queries to fetch all records. Maps the Entity objects into a 2D Object array (Object[][]) matching the table columns and updates the View.

Data Access Layer (DAO) DAO

Classes responsible for direct interaction with the database using Hibernate Session.

ActivityDAO.java

Method Signature Precise Functionality
insertActivity(Session session, Activity activity) Persists a transient Activity instance into the database using session.persist().
isTrainerOccupied(...) Executes an HQL COUNT query to check if a specific trainer is already assigned to an activity on the given day and hour.
getMaxActivityCode(Session session) Executes a query sorting Activity IDs in descending order and limits results to 1 to find the current highest ID.
findAllActivities(Session session) Executes a generic HQL query SELECT a FROM Activity a to return a List of all registered activities.
getActivityStatisticsById(...) Performs a LEFT JOIN between Activity and Client sets. Returns an Object[] with Name and Client Count.

ClientDAO.java

Method Signature Precise Functionality
existDNI(Session session, String dni) Checks if a client with the specific Personal ID (DNI/PESEL) exists. Used for validation before insertion.
insertClient / updateClient Standard Hibernate persistence methods (persist/merge) for the Client entity.
getAllClients(Session session) Retrieves all rows from the CLIENT table mapped to the Client class.
getMaxMemberNumber(Session session) Returns the string value of the highest mNum currently in the database. Used for ID auto-generation.
getGlobalStatistics(Session session) Aggregates complex data: Total clients, List of birthdates (for avg age), and List of Categories/Prices (for revenue).
getStatisticsForActivity(...) Similar to global stats, but filtered by Activity ID via JOIN to analyze a specific class.

TrainerDAO.java

Method Signature Precise Functionality
insertTrainer / updateTrainer Persists new Trainer entities or updates existing ones in the database.
existTrainerID(Session session, String id) Checks if a Trainer with a specific government ID/DNI (tidNumber) already exists.
getMaxTrainerCode(Session session) Fetches the highest alphanumeric Trainer Code (e.g., "T099") to assist in generating the next sequential code.
findAllTrainers(Session session) Returns a complete list of Trainer entities.

Model Layer (Entities) Model

POJO classes annotated with JPA/Hibernate metadata.

Activity.java

  • Fields: aId (PK), aName, aDescription, aPrice, aDay, aHour.
  • Relationships:
    • @ManyToMany with Client (via PERFORMS table).
    • @ManyToOne with Trainer (mapped by atrainerInCharge).
  • getDisplayName(): Helper returning aName for UI display.

Client.java

  • Fields: mNum (PK), mName, mId (Gov ID), mBirthdate, mPhone, memailMember, mstartingDateMember, mcategoryMember.
  • Relationships: @ManyToMany with Activity (inverse side).
  • NamedQueries: Pre-compiled HQL queries for efficient searching.

Trainer.java

  • Fields: tCod (PK), tName, tidNumber, tphoneNumber, tEmail, tDate, tNick.
  • Relationships: @OneToMany with Activity.
  • toString(): Overridden to return tName for correct display in JComboBoxes.
